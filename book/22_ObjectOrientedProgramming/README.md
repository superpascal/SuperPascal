## Notes

**OOP in SuperPascal:**
- **Hybrid model** — Records for data, classes for behavior
- **Single inheritance** — One parent class
- **Reference semantics** — Classes always heap-allocated
- **Virtual methods** — Runtime polymorphism via vtables
- **Lightweight** — Minimal overhead for retro platforms

**Class vs Record:**
- **Records** — Value types, data containers, fast SoA iteration
- **Classes** — Reference types, behavior orchestration, inheritance

**Use Cases:**
- **Game systems** — Managers, controllers
- **UI components** — Buttons, menus, widgets
- **Scene management** — Scene objects, state machines
- **Behavior layers** — Entity behaviors, AI

**Educational Approach:**
- **Practical focus** — OOP applied to game development
- **Code examples** — Implement classes in SuperPascal
- **Progressive complexity** — Build from simple to advanced
- **Integration** — Connect classes to game systems
